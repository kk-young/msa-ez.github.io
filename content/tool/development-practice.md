<h3>강사는 클래스를 진행하기 전에 다음의 준비과정을 진행할 수 있다:</h3>
1. 수강생 인원이 모두 문제 없이 서비스에 정상적으로 접속하여 출석했는지 확인
2. 수강생의 이름이 정확한지, 중복된 수강생이 존재하는 확인
> [팁] 중복된 수강생이 있으면 강의 진행중 전체 인원이 각 랩을 정상 수행했는지 체크하기가 어려워지기 때문에 강의 초기에 확인해 주는 것이 좋습니다.
3. 수강생이 원래 등록된 인원보다 더 많은 지 확인

  3.1 labs 폴더는 하위에 lab들을 포함한다. 
  - lab 은 클래스 입장시 아래 이미지 처럼 개별 랩으로 만들어진다. lab 폴더명이 파란색 동그라미 안에 그려진 이름으로 표시된다.
  <img width="1290" alt="image1" src="https://user-images.githubusercontent.com/3138379/99624419-a1d9ab00-2a71-11eb-8de9-23e7a32c0930.png">

  - 랩이름이나, 랩에 대한 상세 정보는 Lab_Metadata.json 에 작성한다.

  3.2 Lab_Metadata.json

  ```
  {
      "labName": "[구현] 데이터프로젝션-CQRS",
      // ide, event-storming 를 가진다.
      "tool" : "ide", 
      "labTime" : 10,	
      "labScenario" : "이벤트를 수신하여 READ 모델을 만드시오",
      // event-storming 랩 시작시 실행할 json 스크립트 
      "startArtifact": "start.json",  
      // ide 랩 시작시 템플릿으로 필요한 파일이 있는경우 포함
      "templateFile": "xxx.zip",	
      // 동영상이 있을 경우 포함 
      "video": "https://youtu.be/ipXtftkO5eo",
      // 체크포인트를 넣고 싶을경우 포함한다.
      "checkPoints": [
          {
            "id": "id1",
              "text": "체크포인트 질문하세요",
              "testShell": "test OrderCancelled"
          }
      ],
      // 힌트를 넣고 싶을경우 포함한다.
      "hints": [{
          "text": "",
          "regExp": ""
      }]
  }

  ```
  | 이름 | 설명 | 사용법 |
  |---:|:---:|:---:|
  |`labName`| 랩실의 이름 |  | 
  |`labScenario`| 랩실의 간략설명 |  | 
  |`tool`| 랩실 오른쪽에 나오는 툴의 종류 | ide, event-storming , quiz | 
  |`labTime`| 랩진행 시간 표시 | 10 으로 적으면 10분이라고 표시됨 | 
  |`startArtifact`| event-storming 랩 시작시 실행할 json 스크립트  |  | 
  |`templateFile`| ide 랩 시작시 템플릿으로 필요한 파일이 있는경우 포함 |  | 
  |`video`| 랩실에 동영상을 보여주고 싶을경우 설정 | https://youtu.be/xxx 주소 | 
  |`checkPoints`| 화면상에 체크포인트를 넣을때 설정 | [{}, {}] 형식 | 
  | | checkPoints 내부 설정  |  | 
  | id | 내부 아이디값  | 한개의 랩에서 고유값을 가짐 | 
  | text | 체크포인트 지시사항  |  | 
  | testShell | test.sh 파일내부의 로직 비교  | test.sh 파일이 랩에 존재해야함 | 
  | javascript | js 파일을 실행  | js 파일명을 설정하고, 해당 파일이 랩에 존재해야함 | 
  | regExp | 바로 결과값을 비교하여 true, false 값을 체크함  | "HTTP/1.1 201" 이렇게 설정시 터미널에 HTTP/1.1 201 값을 체크 | 
  |`hints`| 화면상에 힌트를 넣을때 설정 | [{}, {}] 형식  |
  | | hints 내부 설정  |  | 
  |text| 힌트로 보여줄 문구 | |
  |javascript| 특정상황에서 힌트를 노출시킬때 설정 | return true 값으로 설정하면 보여짐|

  4. classes 폴더
  - 하위에 running 폴더를 포함해야한다. 
  - 같은 course 에는 여러개의 클래스가 있을수 있다. 랩 순서나, 기간등이 다를수 있기때문에 해당 정보를  Class_Metadata.json 에 기입한다.

  4.1 Class_Metadata.json
  ```
  {
    // 강사와 수강생은 UI 가 약간씩 다르다. 강의를 진행할 email 을 적는다.
    "teacherId": "admin@uengine.org",
    "className": "msaez 에 클래스 생성방법",
    "description": "강의의 설명을 작성하는 공간입니다.",
    // 시작날자와 종료일에 따라서, 수강중인강의, 완료된 강의로 구분된다.
    "classStartDate": "2020.11.10",
    "classEndDate": "2020.11.13",
    // 특정 키 값을 주어서 키값을 입력해야만 강의에 접속이 가능하다.
    "connectionKey": "msaez",
    // openClass가 true 일 경우 누구나 connectionKey값을 알면 들어올 수 있다.
    // openClass가 false 일 경우 groupedUsers 에 적혀있는 user 만 입장이 가능하다.
    "openClass": true,
    "groupedUsers":[
      {"groupName": "Mercury", "users": [
          {"name": "홍길동", "email": "gdhong@uengine.org"}                                
      ]}
    ],
    // 클래스에 포함시킬 랩들을 배열형식으로 작성한다. 배열 순서대로 랩이 나타난다.
    "labsList": [
      "1-make-folder"
      ,"2-deploy-to-msaez"
      ,"3-make-checkpoint"
    ],
    // classId 와 courseId 는 클래스와 코스의 폴더명과 일치시킨다.
    "classId": "guide",
    "courseId": "make-class-guide",
    // 해당 클래스의 상위코스에 대한 정보를 간략하게 적는다.
    "course": {
      "instructorName": "유엔진",
      "instructorId": "admin@uengine.org",
      "courseName": "CNA 분석,개발,운영",
      "courseDesc": "클라우드 네이티브 실습"
    }
  }
  ```

  | 이름 | 설명 | 사용법 |
  |---:|:---:|:---:|
  |`teacherId`| 강의를 진행할 이메일 | 이메일 값 | 
  |`className`| 메인화면에서 보여질 클래스 이름 |  | 
  |`description`| 메인화면에서 보여질 클래스간략 설명 |  | 
  |`classStartDate`| 시작일 | "2020.11.10" | 
  |`classEndDate`| 종료일 | "2020.11.13" | 
  |`connectionKey`| 특정 키 값을 주어서 키값을 입력해야만 강의에 접속이 가능하다 | 설정안하면 누구나 접속가능 | 
  |`openClass`| true 일 경우 누구나 접속가능, false 일 경우 groupedUsers 에 적혀있는 user 만 입장이 가능| 기본값 false | 
  |`groupedUsers`| openClass값이 false일때 접속할수 있는 유저목록을 관리 | `[{"groupName": "1team", "users": [{"name": "홍길동", "email":"gdhong@uengine.org"} ]}]` | 
  |`labsList`| 클래스에 포함시킬 랩 리스트 | 배열형식으로 작성| 
  |`classId`| 클래스 고유 아이디 | 클래스 폴더명과 일치 | 
  |`courseId`| 코스 고유 아이디 | 코스 폴더명과 일치 |

### 2. 배포 방법

<h3>랩실 배포</h3>

  msaez 클래스는 minio 에서 파일을 관리한다.  
  minio 에 파일을 업로드만 시키면 클래스 리스트에서 나타나게 된다.  


<h3>1. minio client 설치</h3>
  - https://docs.min.io/docs/minio-client-quickstart-guide.html
  - 터미널을 열어서 아래 명령어를 실행하여 minio client 설치한다.
  - 현재 폴더 기준으로 설치를 하는 것이기 때문에 배포하려는 최상위 경로에서 설치를 한다.
  ```
  wget https://dl.min.io/client/mc/release/linux-amd64/mc
  chmod +x mc
  ./mc --help
  ```

<h3>2. minio client와 minio 연결</h3>
  - [minio서버주소:port] [minioID] [minio비번] 는 강사에게 개별로 전달
  ```
  ./mc config host add minio http://[minio서버주소:port] [minioID] [minio비번]
  ```

<h3>3. 파일 업로드</h3>
  - 혹시나 실수로 파일을 업로드 하는일이 없도록 한다.

  - 랩실의 예제 코드에서 classes - running - guide - Class_Metadata.json 의 파일을 열어서 className 을 임의로 변경하고, 배포를 진행한다.
  - 파일이 정상적으로 저장되었는지 확인한다.
  ```
    "className": "test 로 배포",
  ```

  - 아래 명령어는 make-class-guide 폴더 내용 전체를 labs-msaez.io/test 폴더에 복사하는 명령어이다.

### 3. 체크포인트 만드는 방법

  체크포인트는 랩별로 생성이 가능하다.  
  랩에 존재하는 Lab_Metadata.json 파일에 아래와 같은 형식으로 입력만 하면 체크 포인트 문구가 나타난다. 이제 체크를 하기 위해서는 약간의 로직을 추가하면 된다.  
  tool 별로 체크포인트를 설정하는 방법이 약간씩 다르기 때문에 툴별로 나누어서 설명을 진행한다.  
  ```
  "checkPoints": [
          {
            "id": "id1",
              "text": "체크포인트 질문하세요"
          }
      ]
  ```

<h3>tool : ide</h3>

<h3>1. regExp</h3>
  - 가장 기본적인 사용법은 regExp 를 사용하여 결과 제출시 작업하고 있는 터미널을 그대로 가지고와서 체크하는 방법이다.
  - 아래와 같이 설정을 한 후, 터미널을 연 후, http www.example.com 을 조회한다.
  - http 명령의 결과로 HTTP/1.1 200 OK 값이 나왔을 텐데, 해당 문구를 바로 비교하여 체크를 한다.
  ```
  "checkPoints": [
          {
            "id": "id1",
              "text": "regExp 를 사용하여 데이터를 조회한 결과를 체크하시오",
              "regExp": "HTTP/1.1 200"
          }
      ]
  ```

<h3>2. testShell</h3>
  - 이 설정은 같은 랩실에 test.sh 파일이 존재해야 작동한다.
  - test.sh 에 쉘 스크립트로 로직을 설정 한 후, 결과값을 비교하여 작동한다.
  - "testShell": "test [키값]" 방식으로 Lab_Metadata.json 에 설정을 하게 된다.
  - 여기서 [키값]은 test.sh 에 설정되어있는 키 값이다.
  ```
  "checkPoints": [
          {
            "id": "id2",
              "text": "testShell 를 사용하여 데이터를 조회한 결과를 체크하시오",
              "testShell": "test check1"
          }
      ]
  ```
  - 이렇게 설정 후 같은 랩실에 test.sh 파일을 만든 후 아래와 같이 설정한다.
  - test.sh 파일은  ####### (#7개) 를 체크한 후에 test check1 이라는 문구를 인식 한 후, 결과값이 HTTP/1.1 200 이 있는지를 체크하는 로직이다.
  - 다음 라인에 나오는 부분이 실행하는 명령어이다.
  - echo "####### finish" 를 test.sh 파일의 가장 하단에 위치하여 테스트가 끝났다는것을 확인시켜준다.
  ```
  echo "####### test check1 : HTTP/1.1 200"
  http www.example.com
  echo "####### finish"
  ```
  - 만약 체크를 할것이 두개라면 ,(콤마) 로 구분하면 두가지 조건이 모두 충족되었을때 체크박스에 체크가 된다
  ```
  echo "####### test OrderCancelled : \"orderStatus\": \"OrderCancelled\" "
  http localhost:8090/orderStatuses
  echo "####### test DeliveryStarted : \"orderStatus\": \"OrderCancelled\",\"deliveryStatus\": \"DeliveryStarted\" "
  http localhost:8090/orderStatuses
  echo "####### finish"
  ```

<h3>tool : event-storming</h3>
  event-storming 으로 사용을 할때는 화면상의 내용을 javascript 로 체크를 한다.  
  체크를 할적에는 inline 으로 스크립트 코드를 바로 작성 할 수도 있고, js 파일을 만들어서 해당 파일을 바로 실행 시키는 방법이 있다.  

  ```
  "checkPoints": [
          {
              "id": "id3",
              "text": "inline javascript code 를 체크한다.",
              "javascript": "oneOf(t.query('$.Event[*].name'), ['OrderPlaced', 'OrderCanceled'])"
          },
          {
              "id": "id4",
              "text": "파일로 체크포인트를 만드는 방법을 확인한다.",
              "javascript": "isPostPoned.js"
          }
      ],
  ```
<h3>1. inline javascript code</h3>
  - 위의 checkPoints 'id3' 을 보면 inline code 가 들어가 있다.  
  - 이벤트를 2개 만들고 이벤트 이름에 'OrderPlaced' 와 'OrderCanceled' 를 넣은 후에 왼쪽 상단에 제출 버튼을 클릭한다.
  - t.query('$.Event[*].name') 부분에 의해서 화면상의 이벤트를 모두 가져온 후에, 값을 비교하여 정상적으로 들어가 있을 경우 체크박스를 체크하게 된다.

<h3>2. make javascript flie</h3>
  - 위의 checkPoints 'id4' 를 보면 js 파일명이 들어가 있다. 이때 isPostPoned.js 파일은 랩실안쪽에 들어가 있어야한다.
  - 가능하면 파일명은 해당 결과를 직관할수 있는 명칭으로 만들어 주면 재사용성면에서 편리하다.  
  - js 파일의 내용은 아래와 같다.
  - result.elements 의 값은 화면상의 이벤트 스토밍한 결과값이고, 그중에서 이벤트를 찾아서 p.p 형식으로 되어있는지 체크하는 로직이다.
  - 마지막에 returnValue 부분에 최종 결과를 true, false 형식으로 보여주면 된다.
  - 이벤트를 2개 만들고 이벤트 이름에 'OrderPlaced' 와 '주문됨' 을 넣은 후에 왼쪽 상단에 제출 버튼을 클릭 하여 체크포인트가 작동하는지 확인한다.  

  ```
  var count = 0;
  var postponed = 0;

  Object.keys(result.elements).forEach(key => {
      var item = result.elements[key];
      if (item && item._type == 'org.uengine.modeling.model.Event') {
          count++;
          if (item.name.endsWith('d') || item.name.endsWith('됨') || item.name.endsWith('함')) postponed++;
      }
  });

  returnValue(count == postponed);

  ```


### 4. 퀴즈 만드는 방법

  퀴즈는 랩실에 quiz 라는 폴더를 만들어서 구성 할 수있다.    
  quiz 폴더에는 각각 퀴즈에 해당하는 html 이 존재한다.  
  해당 html 들을 만든 후 Lab_Metadata.json 파일에서 "tool" : "quiz" 라는 옵션을 주어서 퀴즈 랩을 만들 수 있다.

<h3>quiz.html 구성</h3>

  1. 지문을 드래그 하여 값을 매칭시키는 방법이 있다.

  - <placeholder answer = "main"/> 에 <answer> 태그에 해당하는 정답을 드래그로 넣을수 있도록 구성되어져있다.

  ```
  <v-card class="card1">
      <answer  data="main"/>
      <answer  data='out'/>
      <answer  data="."/>
  </v-card>

  <pre>

  static void <placeholder answer = "main"/> {
      System.<placeholder answer = "out"/>.println();
  }
  </pre>     

  ```
  - 이렇게 설정되어있는 문제의 체크포인트는 아래와 같이 구성된다.
  - <placeholder> 로 되어있는 모든 값이 매칭이 되었을때 체크포인트의 체크박스가 체크된다.
  ```
  "checkPoints": [
          {
              "text": "자바 문법 기본",
              "javascript": "result.quiz['quiz1'].allRight"
          }
      ]
  ```


  2. 체크박스로 문제를 푸는 방법이 있다.
  - 아래는 1번문제 체크포인트에 해당하는 html 파일 내부 구조이다.
  - <check right> 태그에 해당하는 값이 정답이다.
  ```
  [질문] Java 언어로 개발되어있는 어플리케이션을 패키징하여 실행가능한 test.jar 파일로 생성이 되었다.
  다음중 해당 파일을 CLI 에서 실행할 수 있는 방법은?

  [답변]
  <check>1. javac test.jar</check>
  <check right>2. java -jar test.jar</check>
  <check>3. ./test.jar</check>
  <check>4. exec test.jar</check>
  ```

  - 이렇게 설정되어있는 문제의 체크포인트는 아래와 같이 구성된다.  
  - .score > 0 로 체크를 하여 정답이 잘되어있는지 체크한다.  
  ```
  {
      "text": "1번문제",
      "javascript": "result.quiz['quiz4'].score > 0"
  }
  ```

## Theia IDE 기반 Lab 의 생성

***

![msaez-logo](https://user-images.githubusercontent.com/487999/91138051-d267ae00-e6e9-11ea-918a-bd0390412d4b.png)

<h3>Minio 파일 다운로드</h3>

Minio에 접속해서 http://35.200.116.236:9000/minio/labs/YHtest/ 

YHtest폴더를 다운로드 받습니다.

<h3>Class 등록</h3>
VS Code로 YHtest 폴더를 열은 다음 Class_Metadata.json 파일을 클릭합니다.

```{
    "teacherId": "",
    "className": "Test Course",
    "description": "Test를 위한 과정",
	"classStartDate": "2020.08.25",
    "classEndDate": "2020.08.25",
    "openClass": true,
    "groupedUsers":[
    ],
    "labsList": [
        "1"
    ]
}
```
teacherId 는 강사 ID 이므로 강사의 gmail 주소를 적고,

className 에는 개설하고자 하는 강좌의 이름, description 에는 강좌의 대한 설명,

classStartDate 에는 강좌의 시작 날짜, classEndDate 에는 강좌의 종료 날짜,

"labsList"에는 랩의 개수 만큼 목록을 작성합니다. 현재는 랩이 한개 밖에 없기 때문에 "1"만 표기합니다.

"1"에 대한 실습 파일은 labs/1 에 작성됩니다.

<h3>Template 만드는 방법</h3>

labs/1 에 New File을 생성하고 이름은 HelloWorld.java 로 생성합니다.
```
public class HelloWorld{
    public static void main(String args[]){
     
    }
}
```
HelloWorld.java 내용을 위와 같이 작성합니다. 기본 파일 프로젝트가 없기 때문에 템플릿으로 만드는 것입니다.

작성을 완료한 후에는 HelloWorld.java가 있는 경로에 가서 HelloWorld.zip 파일로 압축 시킵니다.

<h3>test.sh 을 통한 테스트의 자동화와 체크포인트 연동</h3>

labs/1 에 test.sh 파일을 만든 후 컴파일을 시키기 위해 test.sh 파일에 아래와 같이 작성합니다.
```
echo "####### test: Hello, world!"
javac HelloWorld.java
java HelloWorld
echo "####### finish"

```

<h3>Lab 등록</h3>

labs/quiz에 들어가서 Lab_Metadata.json 파일을 클릭합니다.
```
{
    "labName": "Hello World",
    "tool" : "ide",
    "labTime" : 10,
    "labScenario" : "print Hello World",
    "templateFile": "HelloWorld.zip",
    "checkPoints": [
        {
            "text": "Hello World를 출력하세요",
            "testShell": "test"
        }
    ],
    "hints": [
		{
        "text": "Hello World"
        }
	]
}

```
labName은 실습하려는 실습명을 적고 tool은 개발환경이 필요하기 때문에 ide 라고 적습니다.

labTime은 실습 시간, labScenario는 시나리오에 대해 적어줍니다.

templateFile은 위에서 작성했던 템플릿을 불러오기 위해 "templateFile": "HelloWorld.zip" 을 적어줍니다.

checkPoints 에는 앞서 작성했던 test.sh 파일을 불러오기 위해 "testShell": "test"을 적어줍니다.

<h3>Minio 반영</h3>
수정된 파일을 반영하기 위해 저장한 후에 저장된 경로로 디렉토리 이동 후 아래와 같이 리눅스 명령어를 타이핑 합니다.

```
mc cp --recursive . minio/labs/YHtest
```
